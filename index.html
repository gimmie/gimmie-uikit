UI style guides

This document is created to define a consistent user interface for all Gimmie's products and websites.


Colors

  • Body Background color always is “#f1f1f1”

  • All the colours we use should select from our “Color Swatches”

Buttons

All buttons will get “btn” class by default and also for sizes will follow the bootstrap default classes.

  • All the “call to action” button types should get “btn-gimmie” class

  • All the “info” button types should get “btn-info” class

  • All the “save, update, upgrade, success, send” button types should get “btn-success” class

  • All the “sign up, sign in” button types should get “btn-warning” class

  • All the “cancel, close, dismiss” button types should get “btn-default” or “btn-link” class, depends on the place

  • All the “delete, remove” button types should get “btn-danger” class

  • Buttons should not get any extra margin at all!

Typography

  • We should not overwrite any additional font size for any element. Font sizes are defined in bootstrap

  • For putting a leading text we use “lead” class for “p” tag

Navbar

  • for fixed navbar we should use “navbar-fixed-top” and “navbar-gimmie” classes

  • None of dropdown or normal buttons inside the navbar will give any button class

jQuery plugins

  • Font Awesome for icons instead of Glyph icon which is bootstrap default icon font, see the comparison here. There is a gem for FontAwesome in RoR, here is the Gem link.

  • iCheck plugin for checkboxes and radio buttons, for more information please visit it's page.
